# Geometry Store

`GeometryStore` is the central geometry state manager. It holds all shapes, syncs to the DCEL, and integrates with history for undo/redo.

## What It Owns

- **Completed shapes**: `polygons`, `rectangles`, `ellipses`, `constraints` (each an `Array`)
- **Working (transient) shapes**: `workingPolygon`, `workingRectangle`, `workingEllipse`, `workingConstraints` -- these represent shapes being actively drawn, NOT persisted
- **DCEL index**: `dcelIndex: DCELShapeIndex` for planar subdivision queries

All shapes use IDs with prefixes: `ply_` (polygon), `rct_` (rectangle), `elp_` (ellipse), `cns_` (constraint). IDs are generated by `HistoryManager.generateStableId()` which uses UUIDs plus a monotonic counter to prevent collisions across load/save cycles.

## Direct/Public Method Pattern

Every CRUD method comes in two forms:

| Public (records history) | Direct (no history recording) |
|--------------------------|-------------------------------|
| `addPolygon(template)` | `addPolygonDirect(polygon)` |
| `deletePolygon(id)` | `deletePolygonDirect(id)` |
| `updatePolygon(id, fn)` | `updatePolygonDirect(id, fn)` |
| `setPolygonFillColor(id, color)` | `setPolygonFillColorDirect(id, color)` |
| (same pattern for rect/ellipse/constraint) | |

The public method calls Direct internally, then records the operation to HistoryManager. HistoryManager uses only Direct methods during undo/redo replay to avoid infinite recording loops.

Direct methods still emit change events (`polygonsChanged`, `rectanglesChanged`, etc.) so the UI stays responsive during undo/redo. They just don't loop back into history.

## DCEL Sync

- **Add/delete**: DCEL is updated immediately (synchronously).
- **Update** (during drag): DCEL sync is **debounced per shape ID by 200ms** (via lodash). This keeps the hot path fast during rapid mouse movement.
- **Immediate flag**: The `reconstrain()` method bypasses debounce with `immediate=true` so constraint solving results appear instantly.

## Events

Emitted on `EventEmitter`: `polygonAdded`, `polygonsChanged`, `rectangleAdded`, `rectanglesChanged`, `ellipseAdded`, `ellipsesChanged`, `constraintAdded`, `constraintsChanged`, plus `workingXxxChanged` variants for all working shapes.

## Key Query Methods

- `getAllGeometryIds(): Set<Id>` -- all completed shape IDs
- `getAllGeometryAsSegments()` -- all shapes converted to segment form (for intersection queries)
- `getMaxRenderOrder(): [number, number]` -- max render order and count at that value
- `resolveConstraintEndpoint(endpoint): SheetPosition` -- resolves a locked endpoint by looking up the referenced shape and computing the relevant point

## Constraint Endpoint Resolution

`resolveConstraintEndpoint(endpoint)` handles four endpoint types:
- `"point"` -- returns the literal `SheetPosition`
- `"locked-rectangle"` -- looks up rectangle, computes corner via `rectCorners()`
- `"locked-ellipse"` -- looks up ellipse, computes key point via `ellipsePoints()`
- `"locked-polygon"` -- looks up polygon, returns the indexed segment's point
